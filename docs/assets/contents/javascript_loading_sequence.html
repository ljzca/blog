Example html page:
<!DOCTYPE html>
    <html>
      <head>
        <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.3.1/jquery.min.js"></script>
        <title>Page Title</title>
        <script>
          $(document).ready (function(){
            console.log('JQuery dom ready')
          });
        </script>
        <script>
          document.addEventListener("DOMContentLoaded", function() {
            console.log('ready')
          });
        </script>
        <script>
          console.log('head')
        </script>
      </head>
      <body>
        <script>
          console.log('begin')
        </script>
        random text...
        <img src="http://randomsite">
        <script>
          console.log('end')
        </script>
      </body>
    </html>
console log:
index.html:17 head
    index.html:22 begin
    index.html:27 end
    index.html:13 ready
    index.html:8 JQuery dom ready
    index.html:25 GET http://randomsite/ net::ERR_NAME_NOT_RESOLVED
JavaScript will always be executed from top to bottom when the page is loading (including event listener registration). JQuery document ready is always triggered after DOMContentLoaded. Page rendering is done after page loading (maybe done asynchronously) because the rendering error can be thrown after document ready event.

P.S. With break point debugging, when stopping at console.log('head'), it is very clear to see that the page body is not loaded yet.
